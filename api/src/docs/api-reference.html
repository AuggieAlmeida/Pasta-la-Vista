Auth Endpoints

Registro, login e gerenciamento de sessoes via JWT. Access tokens expiram em 15 minutos, refresh tokens em 7 dias.

POST /api/v1/auth/register Registrar novo usuario
Publico
▶

Cria um novo usuario com role CLIENT. Retorna access e refresh tokens. A senha deve ter min 8 chars, 1 maiuscula, 1 minuscula e 1 numero.

CampoTipoDescricao
name*stringNome completo (3-100 chars)
email*stringEmail unico
password*stringSenha forte (min 8 chars)
phonestringTelefone (opcional)
Exemplo Request 
{
  "name": "Maria Silva",
  "email": "maria@email.com",
  "password": "Senha123!",
  "phone": "+5511999991111"
}
201 Created 400 Validation 409 Duplicate
POST /api/v1/auth/login Login
Publico
▶

Autentica com email e senha. Retorna access token (15min) e refresh token (7d).

Exemplo Request 
{
  "email": "maria@email.com",
  "password": "Senha123!"
}
Exemplo Response 200 
{
  "status": "success",
  "data": {
    "user": { "id": "uuid...", "name": "Maria Silva", "role": "CLIENT" },
    "access_token": "eyJhbG...",
    "refresh_token": "eyJhbG..."
  }
}
200 OK 400 Validation 401 Unauthorized
POST /api/v1/auth/refresh Renovar tokens
JWT
▶

Usa o refresh token para obter novos access e refresh tokens. O refresh token anterior e invalidado.

200 OK 401 Unauthorized
POST /api/v1/auth/logout Logout
JWT
▶

Remove a sessao do Redis, invalidando o refresh token.

200 OK 401 Unauthorized
Cardapio

Menu Endpoints

Consulta de produtos com cache Redis. Endpoints publicos para leitura, admin-only para escrita.

GET /api/v1/menu Listar cardapio
PublicoCache 5min
▶

Retorna todos os produtos ativos ordenados por categoria e nome. Resultado cacheado no Redis (key: menu:all) por 5 minutos.

200 OK
GET /api/v1/menu/search?q={query} Buscar produtos
Publico
▶

Busca por nome usando regex case-insensitive. Limite de 20 resultados. Sem cache para resultados real-time.

ParamTipoDescricao
q*query stringTermo de busca (min 1 char)
200 OK400 Missing Query
GET /api/v1/menu/category/{category} Filtrar por categoria
PublicoCache 5min
▶

Categorias validas: pizzas, bebidas, sobremesas, massas, aperitivos

200 OK
GET /api/v1/menu/{id} Detalhe do produto
PublicoCache 10min
▶

Retorna detalhes completos incluindo customizacoes disponiveis. id e um MongoDB ObjectId.

200 OK404 Not Found

Pedidos

Order Endpoints

Criacao e consulta de pedidos. Todos os endpoints requerem autenticacao. Precos sao calculados server-side.

POST /api/v1/orders Criar pedido
JWT
▶

Valida produtos, verifica estoque, calcula precos server-side, salva em PostgreSQL + MongoDB, decrementa estoque. Delivery: R$ 5,00 fixo.

Exemplo Request 
{
  "items": [{
    "product_id": "507f1f77bcf86cd799439011",
    "quantity": 2,
    "customizations": [{ "customization_id": "cust1", "price_modifier": 10 }]
  }],
  "address": {
    "street": "Rua das Flores", "number": "42",
    "city": "Sao Paulo", "state": "SP", "zip": "01234-567"
  },
  "notes": "Sem cebola",
  "payment_method": "PIX"
}
201 Created400401422 Stock
GET /api/v1/orders Listar meus pedidos
JWT
▶

Retorna apenas os pedidos do usuario autenticado, ordenados por data decrescente.

200 OK401

Modelos

Data Schemas

Schemas de dados usados nos requests e responses da API.

Product
Produto do cardapio armazenado no MongoDB
  • _id ObjectId
  • name string
  • description string
  • price number
  • category enum
  • customizations Customization[]
  • preparation_time integer (min)
  • active boolean
Order
Pedido armazenado em PostgreSQL + MongoDB
  • id UUID
  • userId UUID
  • items OrderItem[]
  • subtotal number
  • delivery_fee number (R$ 5)
  • total number
  • status enum (6 states)
  • payment_method PIX|CARD|CASH
Customization
Opcao de customizacao de produto
  • _id ObjectId
  • type size | ingredient
  • name string
  • price_modifier number (R$)
  • available boolean
Address
Endereco de entrega
  • street string
  • number string
  • complement string?
  • city string
  • state string (2 chars)
  • zip XXXXX-XXX
AuthResponse
Resposta de autenticacao
  • user.id UUID
  • user.name string
  • user.email string
  • user.role CLIENT | ADMIN
  • access_token JWT (15min)
  • refresh_token JWT (7d)
ErrorResponse
Padrao de erro da API
  • status "error"
  • message string
  • errors {field, message}[]
